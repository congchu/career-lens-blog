## 들어가며

채용공고를 읽다 보면 자주 보이는 문구들이 있습니다.

> "열정적인 인재를 모십니다"
> "빠르게 성장하는 팀"
> "도전을 두려워하지 않는 분"

좋은 말이죠? 그런데 이런 문구가 **구체적인 내용 없이 반복**되면, 주의해야 합니다.

원티드 프론트엔드 공고 101개를 분석해서, **어떤 표현이 실제 레드플래그이고 어떤 건 괜찮은 건지** 구분하는 법을 정리했어요.

---

## 먼저 오해부터 풀겠습니다

**"열정"이라는 단어 자체가 레드플래그는 아닙니다.**

101개 공고 중 10개(10%)가 "열정"을 언급했는데요. 이 중에는 여기어때, 루루랩, 랭디 같은 괜찮은 회사도 포함되어 있어요.

**문제는 "열정"이 구체적인 자격요건을 대체할 때입니다.**

| 괜찮은 사용 | 위험한 사용 |
|------------|------------|
| "React 개발 3년 + 열정적인 분" | "열정적인 인재를 찾습니다" (기술 요건 없음) |
| "성장하고 싶은 분 (구체적 성장 경로 제시)" | "빠르게 성장하는 환경" (뭐가 빠른지 안 알려줌) |
| "도전적인 기술 과제 (구체적 설명)" | "도전을 두려워하지 않는 분" (뭘 도전하는 건지?) |

---

## 진짜 레드플래그 5가지

### 1. "열정"으로 자격요건을 대체하는 공고

> "개발 문화를 함께 만들어가고자 하는 열정을 가지신 분"

이 문장이 **자격요건의 전부**라면 위험해요. 번역하면:

- "개발 문화가 아직 없다" → 내가 만들어야 한다
- "열정 = 무한 야근에 대한 긍정적 태도"일 수 있음
- 구체적으로 뭘 하는지 안 알려주는 건, **본인들도 모르기 때문**일 수 있음

**판별법**: 기술 자격요건이 구체적인데 "열정"은 추가로 언급 = 괜찮음. 기술 요건이 모호한데 "열정"만 강조 = 주의.

### 2. "빠르게" + "성장" 콤보

101개 공고 중 **62개(61%)**가 "성장"을, **40개(40%)**가 "빠르게"를 언급합니다. 거의 모든 공고가 쓰는 말이에요.

문제는 이겁니다:

> "빠르게 성장하는 스타트업에서 함께할 인재"

이게 뜻하는 것:
- (좋은 경우) 회사가 진짜 빠르게 성장 중 → 기회 많음
- (나쁜 경우) 사람이 부족해서 한 명이 여러 몫 → 소진

**판별법**: "빠르게 성장"과 함께 **구체적인 성과**(MAU, 매출, 투자)가 있으면 괜찮음. "빠르게 성장" + "도전" + "열정"만 있고 숫자 없으면 주의.

### 3. "풀스택" 또는 "멀티" 요구

101개 중 **23개(23%)**가 "풀스택"을 언급합니다.

> "프론트엔드 개발자를 뽑는다면서 왜 백엔드도 하라는 거지?"

풀스택 요구 자체는 트렌드이긴 한데(AWS 18%, Docker 4%), **"프론트엔드 채용"이면서 사실상 "다 해야 하는" 공고**는 주의해야 해요.

| 괜찮은 풀스택 | 위험한 풀스택 |
|-------------|-------------|
| "프론트 중심 + 간단한 API 개발" | "프론트, 백, 인프라, 디자인 다 해주세요" |
| 기술스택이 명확하게 나열됨 | "필요한 건 다 합니다" |
| 팀에 백엔드 개발자가 별도로 있음 | 개발자가 나 혼자일 것 같은 느낌 |

### 4. 복지를 과도하게 강조하는 공고

복지가 좋은 건 좋은 거지만, **기술 내용보다 복지 설명이 3배 긴 공고**는 의심해볼 필요 있어요.

> "무제한 간식, 최신 장비 지급, 자율출퇴근, 리모트 가능, 연 2회 워크샵..."

이게 자격요건보다 길면:
- 기술적으로 매력적인 점이 없어서 복지로 어필하는 것일 수 있음
- 또는 진짜 복지가 좋은 건데, **기술 내용이 부실한 건 별개의 문제**

**판별법**: 복지 + 기술 모두 구체적이면 좋은 회사. 복지만 구체적이고 기술 설명이 "React로 서비스 개발" 한 줄이면 주의.

### 5. "회사 소개"가 공고의 80%인 경우

어떤 공고는 회사 자랑이 800자, 실제 업무 설명이 100자입니다.

> "우리는 시리즈B 투자를 유치한 혁신적인 AI 스타트업으로... (중략 500자) ...세상을 바꾸는 여정에 함께하세요."
> **주요업무**: React로 서비스 개발

이건 **"뭘 하는지는 중요하지 않고 우리 회사에 오는 게 중요하다"**라는 뜻이에요. 구체적인 업무 설명이 없으면, 입사 후에 "이거 하세요 저거 하세요"가 될 확률이 높습니다.

## 레드플래그 vs 그린플래그 비교

| 레드플래그 | 그린플래그 |
|-----------|-----------|
| "열정적인 인재" (기술 요건 없이) | "React, TS 경험 + 성장 의지" |
| "빠르게 성장하는 환경" (숫자 없이) | "MAU 50만, 전년 대비 3배 성장" |
| "필요한 건 다 합니다" | "프론트 중심, BFF API 일부 담당" |
| 복지 500자 + 업무 100자 | 업무 상세 + 기술 스택 명시 |
| "자율적인 분위기" (설명 끝) | "코드리뷰 필수, 주 1회 기술 세미나" |

## 공고에서 좋은 신호 찾기

레드플래그만 말하면 불안해지니까, **"이 공고는 괜찮겠다" 싶은 그린플래그**도 정리해드릴게요.

### 기술 관련 그린플래그
- 기술 스택이 **버전까지 명시** (React 18, Next.js 14 App Router)
- "이런 기술적 도전을 합니다" — **구체적인 문제 설명**
- 기술 블로그 링크 포함
- 오픈소스 기여 언급

### 조직 관련 그린플래그
- "프론트엔드 팀 N명" — **팀 규모 명시**
- "코드 리뷰 문화", "주간 기술 공유" — **개발 문화 구체적**
- "1:1 미팅", "온보딩 프로그램" — **신입 지원 체계**

### 데이터로 본 그린플래그
이전 분석에서 신입을 뽑는 12개 공고의 공통점:
- 80%가 스타트업 (50명 이하)
- 9/12가 퇴사율 5% 이하
- 기술 스택이 비교적 구체적

## 실전: 공고 읽기 연습

아래 공고 문구를 보고, 레드/그린을 판별해보세요.

### 문구 1
> "React, TypeScript 기반 서비스 개발 경험 3년 이상. React Query를 활용한 서버 상태 관리 경험. 코드 리뷰 문화에 익숙한 분."

**그린**. 기술이 구체적이고, 개발 문화를 언급하고, 구체적 도구(React Query)를 명시.

### 문구 2
> "글로벌 시장을 선도하며 혁신을 이끌어갈 열정 있는 인재를 모십니다. 새로운 도전을 두려워하지 않는 분."

**주의**. 열정 + 도전만 있고 구체적 기술/업무 설명 없음.

### 문구 3
> "빠르게 성장하는 환경에서 프론트엔드 전반을 담당합니다. React 경험 필수. MAU 30만 서비스."

**혼합**. "빠르게 성장"이 있지만 MAU 숫자가 있어서 구체적. 다만 "전반을 담당"이 풀스택 의미인지 확인 필요.

---

## 정리: 공고 레드플래그 체크리스트

공고를 읽을 때 아래 항목을 확인하세요:

- [ ] 기술 자격요건이 **3개 이상 구체적으로** 나열되어 있는가?
- [ ] "열정", "도전", "성장"이 **기술 설명 없이 단독으로** 쓰이지 않는가?
- [ ] 주요 업무가 **구체적인 기술/제품 기반**으로 설명되어 있는가?
- [ ] 복지 설명보다 **업무/기술 설명이 더 길거나 같은가?**
- [ ] 회사 소개가 공고의 **50% 이하**인가?

3개 이상 통과하면 괜찮은 공고입니다. 1~2개만 통과하면 더 조사해보세요. 0개면... 패스.
