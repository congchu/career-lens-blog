## 들어가며

이전 편에서 "AI로 이력서 쓰면 티난다"고 했는데요. 그러면 반대로, **AI가 이력서를 평가하면 어떤 결과가 나올까요?**

궁금해서 직접 해봤습니다. 커리어렌즈의 AI 면접관 시뮬레이션에 가상 신입 이력서를 넣고 돌려봤어요.

결과부터 말하면 — **생각보다 칼같고, 생각보다 유용합니다.**

---

## 커리어렌즈 이력서 평가, 뭔가요?

커리어렌즈는 **채용공고 데이터 기반 AI 면접관 시뮬레이션** 서비스입니다.

원리는 이래요:

```
1. 채용공고 URL을 넣는다
2. AI가 공고를 분석해서 "면접관 페르소나"를 생성한다
3. 이력서를 넣는다
4. 면접관이 이력서를 평가하고 피드백을 준다
```

단순히 "이력서 잘 쓰셨네요"가 아니라, **실제 공고의 자격요건과 비교해서** 부족한 점, 강조해야 할 점, 면접에서 나올 질문까지 알려줍니다.

### 어떻게 작동하나? (간단히)

1. **채용공고 크롤링** — 원티드/사람인 등에서 공고 텍스트를 가져옴
2. **면접관 생성** — 공고의 회사 규모, 기술 스택, 문화 등을 반영한 AI 면접관 생성
3. **이력서 파싱** — PDF 이력서에서 텍스트 추출
4. **AI 평가** — Claude API를 사용해 이력서를 다각도로 평가
5. **리포트 생성** — 점수 + 상세 피드백 + 예상 면접 질문

핵심은, 면접관마다 **성격과 관점이 다르다**는 거예요. CTO형 면접관은 기술 깊이를 보고, HR형 면접관은 문화 적합성을 봅니다.

---

## 가상 신입 이력서로 돌려본 결과

가상의 프론트엔드 신입 이력서를 만들어서 테스트해봤습니다.

### 가상 이력서 프로필

- **경력**: 부트캠프 수료, 신입
- **기술스택**: React, JavaScript, CSS
- **프로젝트**: 투두앱, 날씨앱, 개인 블로그
- **특이사항**: TypeScript 경험 없음, 배포 경험 없음

이걸 원티드에서 실제로 올라온 스타트업 프론트엔드 공고에 맞춰서 평가를 돌렸어요.

### AI 면접관이 뭐라고 했나

#### 강점으로 본 것
- React 사용 경험은 기본 요건 충족
- 여러 개의 프로젝트를 만들어본 점은 긍정적

#### 약점으로 지적한 것

> "TypeScript 경험이 없는 것은 큰 약점입니다. 원티드 프론트엔드 공고의 45%가 TypeScript를 요구합니다."

> "세 프로젝트 모두 개인 프로젝트로, 협업 경험을 확인할 수 없습니다."

> "배포 및 운영 경험이 없어, 실무 적용 가능성에 대한 의문이 있습니다."

> "프로젝트 설명에 구체적인 성과 지표가 없습니다. '투두앱을 만들었다'보다 '어떤 문제를 해결했는지'가 중요합니다."

#### 예상 면접 질문

1. "React에서 상태 관리를 어떻게 하셨나요? 프로젝트가 커지면 어떤 문제가 생길 것 같나요?"
2. "TypeScript를 사용하지 않은 이유가 있나요? 타입 시스템의 장점을 어떻게 생각하세요?"
3. "이 투두앱에서 가장 어려웠던 부분은 뭐였나요?"

---

## 피드백을 보고 느낀 점

솔직히 좀 따끔합니다. 하지만 **면접 가기 전에 이걸 알면** 준비를 할 수 있잖아요.

### 이 시뮬레이션이 유용한 이유

**1. "내 이력서가 이 공고에 맞는지" 객관적 판단**

혼자서 이력서를 보면 다 괜찮아 보이거든요. AI가 공고 자격요건과 대조해서 "이게 부족합니다"라고 말해주면 현실을 직시할 수 있어요.

**2. 면접 예상 질문을 미리 준비**

이력서의 약점에서 질문이 나옵니다. 약점을 알면 → 질문을 예상할 수 있고 → 답변을 준비할 수 있어요.

**3. 공고별로 다른 피드백**

같은 이력서도 스타트업 공고 vs 대기업 공고에 넣으면 **평가가 달라집니다.** 스타트업은 "빠르게 배우는 능력"을 보고, 대기업은 "기본기와 깊이"를 보거든요.

---

## AI 평가가 완벽하진 않습니다

당연히 한계도 있어요. 솔직하게 말해야죠.

### AI가 못 보는 것

- **열정과 태도** — 이력서 텍스트만으로는 사람의 에너지를 판단할 수 없음
- **성장 잠재력** — "이 사람이 3개월 후에 얼마나 성장할지"는 AI도 모름
- **문화 적합성의 미묘함** — "우리 팀 분위기에 맞을까?"는 만나봐야 아는 것
- **포트폴리오 품질** — 코드의 깔끔함, 커밋 메시지, PR 문화 등은 별도 확인 필요

### AI 평가를 활용하는 올바른 방법

- ✅ **방향 잡기** — "나한테 뭐가 부족한지" 파악하는 출발점
- ✅ **면접 준비** — 예상 질문으로 리허설
- ✅ **이력서 개선** — 부족한 키워드나 경험 보충
- ❌ 점수에 일희일비하기 — AI 점수가 낮다고 합격 못하는 게 아님
- ❌ AI 피드백만 믿기 — 실제 현직자 피드백도 같이 받으세요

## 직접 해보는 법

커리어렌즈 이력서 평가는 이렇게 쓸 수 있습니다:

### 1. 준비물
- 이력서 (PDF)
- 지원하려는 회사의 채용공고 URL

### 2. 실행
```bash
# 채용공고 URL로 면접관 생성
python main.py crawl --url "채용공고URL"

# 이력서 평가 실행
python main.py evaluate --resume 이력서.pdf --company 회사명
```

### 3. 결과 확인
- 항목별 점수 (기술 역량, 프로젝트, 성장 가능성 등)
- 상세 피드백
- 예상 면접 질문
- 추천 회사 목록

---

## 추천하는 활용법

### 취준 초기: "내 현재 위치" 파악

아무 공고나 3개 골라서 돌려보세요. 공통적으로 지적되는 약점이 **진짜 약점**입니다.

### 이력서 수정 후: "개선 확인"

TypeScript 프로젝트를 추가한 후 다시 돌리면, 점수가 올라가는 게 보여요. 성장을 숫자로 확인할 수 있습니다.

### 지원 직전: "최종 점검"

"이 회사에 내 이력서가 통할까?"를 객관적으로 확인. 부족하면 보충하고, 괜찮으면 자신감을 갖고 지원하세요.

---

## 정리

| 활용 단계 | 하는 일 | 기대 효과 |
|-----------|---------|-----------|
| 현위치 파악 | 공고 3개에 이력서 돌리기 | 공통 약점 발견 |
| 보강 | 약점 기반으로 프로젝트/학습 | 타겟팅된 성장 |
| 재평가 | 수정 이력서 다시 돌리기 | 성장 확인 + 자신감 |
| 최종 지원 | 지원 회사 공고로 맞춤 평가 | 면접 예상 질문 + 약점 보완 |

**AI가 이력서를 평가한다고 해서 무서워할 필요 없습니다.** 오히려 면접관 만나기 전에 미리 피드백을 받을 수 있는 기회예요.

사람 면접관은 "왜 탈락했는지" 안 알려주지만, AI는 알려줍니다. 그게 커리어렌즈의 가치입니다.
