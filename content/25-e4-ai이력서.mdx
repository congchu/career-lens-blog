## 들어가며

ChatGPT에 "프론트엔드 개발자 이력서 써줘" 하면 그럴싸한 문장이 나옵니다.

> "다양한 프로젝트 경험을 보유하고 있으며, 팀 협업을 통해 성과를 창출하는 데 강점이 있습니다."

깔끔하죠? 그런데 문제가 있어요. **채용 담당자가 이 문장을 보면 바로 압니다.** "아, AI로 썼구나."

왜 티가 나는지, 그리고 AI를 **제대로** 활용해서 이력서를 쓰는 방법은 뭔지 — 정리해봤습니다.

---

## AI로 쓴 이력서, 왜 티가 날까?

### 1. "아무 말 대잔치" 증후군

AI가 만든 이력서의 가장 큰 문제는 **구체성이 없다**는 겁니다.

| AI가 쓴 문장 | 왜 안 되는지 |
|-------------|-------------|
| "다양한 프로젝트 경험 보유" | 뭘 몇 개 했는지 모름 |
| "팀 협업을 통해 성과 창출" | 무슨 성과인지 모름 |
| "React를 활용한 웹 개발 경험" | 뭘 만들었는지 모름 |
| "문제 해결 능력이 뛰어남" | 무슨 문제를 어떻게 풀었는지 모름 |
| "새로운 기술을 빠르게 습득" | 증거가 없음 |

채용 담당자는 하루에 수십~수백 개 이력서를 봅니다. AI가 만들어내는 이런 "일반적이고 그럴싸한 문장"은 이미 수없이 봤어요. 2024년부터 AI 이력서가 폭발적으로 늘었거든요.

### 2. 모든 사람이 "같은 말"을 함

ChatGPT에게 이력서를 맡기면, 비슷한 프롬프트를 넣은 사람들은 **비슷한 결과를 받습니다.**

- "성장하는 개발자"
- "사용자 중심의 UI/UX"
- "효율적인 코드 작성을 지향"
- "적극적인 커뮤니케이션"

이런 문구가 10개 이력서 중 7개에 들어있으면, 채용 담당자 눈에는 **전부 같은 사람**으로 보여요.

### 3. "에너지"가 없음

사람이 쓴 이력서에는 약간의 거칠음, 개인적인 표현, 고유한 문체가 있어요. AI가 쓴 이력서는 너무 매끄러워서 오히려 부자연스럽습니다.

> AI: "React와 TypeScript를 활용하여 사용자 경험을 개선하는 프론트엔드 개발에 열정을 가지고 있습니다."
> 사람: "Next.js로 쇼핑몰 만들다가 SSR 삽질 3일 하고, 결국 ISR로 바꿔서 LCP를 2초 줄였습니다."

어떤 게 더 끌리시나요? 두 번째죠. **구체적인 경험에서 나오는 진짜 이야기**가 훨씬 강력합니다.

---

## 그렇다고 AI를 안 쓰라는 건 아닙니다

여기서 오해하면 안 되는 게, "AI로 이력서 쓰지 마세요"가 아니에요. **"AI에게 이력서를 맡기지 마세요"**입니다.

차이를 보여드릴게요.

### 나쁜 사용법: AI에게 "다" 맡기기

```
프롬프트: "프론트엔드 개발자 신입 이력서 써줘. React, TypeScript 할 줄 알아."
```

결과: 누구에게나 적용 가능한 일반적인 이력서 → 티남

### 좋은 사용법: AI를 "도구로" 활용하기

```
프롬프트: "내가 만든 프로젝트 설명을 더 간결하게 다듬어줘.
원문: 'Next.js로 쇼핑몰을 만들었는데, 처음에 getServerSideProps를 
썼다가 너무 느려서 getStaticProps + ISR로 바꿨더니 LCP가 
4.2초에서 1.8초로 줄었습니다'"
```

결과: 내 경험은 그대로, 표현만 더 좋아진 이력서 → 안 티남

**핵심: "내 경험"이 원재료, AI는 "조리 도구"**

## AI를 활용한 이력서 작성법 — 실전 가이드

### Step 1. 먼저 날것 그대로 적기

AI를 열기 전에, **내가 한 것을 있는 그대로** 적으세요.

```
- Next.js 쇼핑몰 만듦
- SSR 느려서 ISR로 바꿈
- LCP 4.2초 → 1.8초
- 상품 목록 무한스크롤 구현
- React Query로 서버 상태 관리
- 디자이너랑 같이 한 건데, 디자이너가 피그마로 주면 내가 퍼블리싱
```

문법 틀려도, 두서없어도 괜찮아요. 이게 **진짜 경험**이니까요.

### Step 2. AI에게 "다듬기"를 요청

날것의 경험을 AI에게 주면서, **다듬기만** 부탁하세요.

```
프롬프트: "아래 내용을 이력서용 문장으로 다듬어줘. 
구체적인 숫자와 기술 이름은 그대로 유지해줘. 
지나치게 매끄럽거나 일반적인 표현은 피해줘."
```

AI 결과:
> "Next.js 기반 이커머스 프로젝트에서 SSR → ISR 전환을 통해 LCP를 4.2초에서 1.8초로 57% 개선. React Query를 활용한 서버 상태 관리 및 무한 스크롤 구현."

이러면 **내 경험 + 깔끔한 표현**이 됩니다.

### Step 3. 공고 맞춤화에 AI 활용

여기가 AI가 진짜 빛나는 부분이에요. 지원하는 회사의 공고를 보고, **내 경험 중 뭘 강조할지** 판단할 때 AI를 쓰세요.

```
프롬프트: "이 채용공고의 자격요건을 분석해서, 
내 경험 중 어떤 항목을 강조하면 좋을지 추천해줘.

[공고 자격요건 붙여넣기]
[내 프로젝트 경험 목록 붙여넣기]"
```

이 방법의 장점:
- 공고마다 강조점을 다르게 → **맞춤형 이력서**
- 내가 놓친 연결고리를 AI가 찾아줌
- 10개 회사에 지원할 때 효율적

## 공고에서 보이는 힌트: "이력서에 이걸 넣으세요"

원티드 프론트엔드 공고 101개를 분석하면, 이력서에 넣어야 할 키워드가 보입니다.

### 자격요건 빈출 키워드 TOP 5

| 키워드 | 등장 비율 |
|--------|----------|
| React | 72% |
| TypeScript | 45% |
| Next.js | 40% |
| 상태관리 (React Query/Zustand) | 19% |
| AWS/클라우드 | 18% |

이력서에 React만 적지 말고, **React로 뭘 만들었는지** 적어야 합니다.

- ❌ "React 사용 경험"
- ✅ "React 18 + TypeScript로 실시간 대시보드 구현, Suspense 활용해 로딩 UX 개선"

### 최근 등장한 새로운 키워드

AI 도구 역량을 요구하는 공고도 15개(15%)나 됩니다. 이력서에 한 줄 넣을 가치가 있어요.

- ✅ "Cursor + Claude Code를 활용한 개발 워크플로우 경험"
- ✅ "AI 도구를 활용해 컴포넌트 개발 속도 2배 향상"

## 채용 담당자가 실제로 보는 것

이력서 심사할 때 채용 담당자가 보는 건 결국 이 3가지입니다:

### 1. "이 사람이 진짜 이걸 했나?"

구체적인 숫자, 기술명, 상황 설명이 있으면 → 진짜구나
일반적인 미사여구만 있으면 → AI가 쓴 거거나, 경험이 얕거나

### 2. "우리 팀에서 쓸 수 있는 사람인가?"

자격요건과 경험이 매칭되는지 — **공고를 잘 읽고 맞춤화한 이력서**가 여기서 차이를 만듭니다.

### 3. "이 사람만의 이야기가 있는가?"

100명이 같은 투두앱을 만들어도, **왜** 만들었고 **어떤 문제**를 만났고 **어떻게** 해결했는지는 다 다릅니다. 그 이야기가 있는 이력서가 살아남아요.

## AI 이력서 체크리스트

이력서를 다 쓴 후, 아래 항목을 확인하세요:

- [ ] **"다양한", "풍부한", "열정적인"** 같은 추상적 형용사가 있나? → 삭제
- [ ] **구체적인 숫자**가 있나? (LCP 몇 초 개선, 사용자 몇 명, 기간 몇 주)
- [ ] **기술명이 정확한가?** (React → React 18, Next.js → Next.js 14 App Router)
- [ ] **내 목소리가 들리는가?** 다른 사람 이력서에 붙여넣어도 말이 되면 너무 일반적인 거
- [ ] **공고 자격요건과 매칭되는가?** 공고마다 강조점이 달라야 함
- [ ] 전체를 소리 내서 읽었을 때 **자연스러운가?** AI 이력서는 소리 내서 읽으면 어색함

## AI 이력서, 써도 될까? — 결론

| | 괜찮은 방법 | 안 되는 방법 |
|---|---|---|
| **작성 주체** | 내가 쓰고, AI가 다듬기 | AI가 쓰고, 내가 복붙 |
| **경험** | 진짜 내 경험 기반 | AI가 만들어낸 가상 경험 |
| **표현** | 구체적 숫자 + 기술명 | "다양한 경험", "열정적인 태도" |
| **맞춤화** | 공고별로 강조점 다르게 | 한 벌로 100군데 지원 |
| **활용 범위** | 문장 다듬기, 맞춤화 분석 | 처음부터 끝까지 생성 |

**AI는 이력서의 "포장"을 도와주는 도구입니다. 하지만 포장할 "내용물"은 여러분이 만들어야 해요.**

이력서에 쓸 경험이 없다고요? 그러면 AI로 이력서를 쓸 게 아니라, AI로 프로젝트를 만들어보세요. **AI 도구를 활용해서 사이드 프로젝트를 만든 과정** 자체가 2026년에는 훌륭한 이력서 소재가 됩니다.
