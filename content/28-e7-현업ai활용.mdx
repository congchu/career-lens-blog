## 들어가며

취준 커뮤니티에서 이런 질문을 정말 많이 봅니다.

> "현업에서는 AI를 진짜 쓰나요?"
> "ChatGPT로 코드 짜면 회사에서 뭐라 안 해요?"
> "AI 도구를 어디까지 써도 되는 건지 모르겠어요"

당연한 궁금증이에요. AI 도구가 엄청 빠르게 발전하고 있는데, 실제 회사에서 어떻게 쓰이는지는 밖에서 보이질 않으니까요.

그래서 **글로벌 개발자 설문 데이터**와 **원티드 채용공고 101개 분석 결과**, 그리고 **국내 기업들의 공개 사례**를 모아서 정리해봤습니다. 현업에서 AI가 어떻게 쓰이고 있는지, 숫자로 보여드릴게요.

## 1. 글로벌 숫자: 개발자 76%가 AI를 쓰고 있다

### Stack Overflow Developer Survey 2024

전 세계 개발자를 대상으로 한 가장 큰 설문입니다. AI 관련 주요 결과:

| 항목 | 수치 |
|------|------|
| AI 도구를 사용 중이거나 사용 예정 | **76%** |
| 현재 실제 사용 중 | **62%** (전년 44%에서 급증) |
| AI 도구에 호의적 | **72%** |
| AI 출력을 신뢰함 | **43%** |
| AI 출력에 회의적 | **31%** |
| AI가 내 직업을 위협한다고 생각 | **30%** |
| AI가 위협이 아니라고 생각 | **70%** |

**핵심 포인트:**

- **62%가 실제 사용 중**입니다. "써봤다"가 아니라 현재 진행형이에요. 전년(44%)에서 18%p나 뛰었습니다.
- 하지만 **신뢰도는 반반** — 43%만 AI 출력을 신뢰합니다. 즉, **쓰긴 쓰지만 무조건 믿지는 않는다**는 뜻이에요.
- **70%가 AI를 위협으로 보지 않습니다.** "AI가 개발자를 대체한다"는 공포는 현업 개발자 사이에서는 소수 의견이에요.

> 📎 출처: Stack Overflow Developer Survey 2024 — AI 섹션
> https://survey.stackoverflow.co/2024/ai

### GitHub Developer Survey

GitHub이 미국 기업 개발자 500명을 대상으로 한 설문:

- **92%**가 AI 코딩 도구를 사용하거나 실험 중
- 생산성 향상을 경험한 비율이 높지만, **"복잡한 작업에는 아직 부족하다"**는 평가도 공존

> 📎 출처: GitHub Blog — Survey reveals AI's impact on the developer experience
> https://github.blog/news-insights/research/survey-reveals-ais-impact-on-the-developer-experience/

## 2. 현업에서 AI를 어디에 쓰나?

Stack Overflow 설문에서 "AI를 어떤 작업에 쓰나?"라는 질문의 답:

| 활용 분야 | 현재 사용 중 |
|-----------|-------------|
| **코드 작성** | 82% |
| **디버깅** | 높음 (구체 수치 비공개) |
| **코드 문서화** | 다음 1년 내 81%가 더 통합 예상 |
| **테스트 코드** | 다음 1년 내 80%가 더 통합 예상 |
| **코드 리뷰** | 증가 추세 |

**가장 많이 쓰는 곳은 "코드 작성"(82%)**입니다. 놀라운 건 아직 사용하지 않는 개발자들이 가장 관심 있는 분야는 **"테스트 코드"(46%)**라는 점이에요.

이걸 현업 맥락으로 번역하면:

### 주니어가 가장 많이 쓰는 방식

1. **보일러플레이트 코드 생성** — 반복적인 CRUD, 폼 컴포넌트 등을 AI로 빠르게 만들기
2. **에러 디버깅** — 에러 메시지를 AI에 넣고 원인 파악
3. **코드 리뷰 보조** — PR 올리기 전에 AI로 먼저 검토
4. **학습** — 모르는 라이브러리/패턴을 AI에게 물어보면서 빠르게 습득

### 시니어가 쓰는 방식

1. **아키텍처 설계 상담** — "이 구조가 확장성이 있을까?" 같은 고수준 질문
2. **코드 리팩토링** — 기존 코드를 AI로 정리/최적화
3. **문서화 자동화** — API 문서, README 등을 AI로 초안 작성
4. **프로토타이핑** — 새로운 기능을 AI로 빠르게 프로토타입 만들고 검증

## 3. 한국 현업은 어떨까? — 채용공고가 말해주는 현실

글로벌 설문은 좋은데, "한국 회사는 어때?"가 더 궁금하시죠. 저희가 분석한 **원티드 프론트엔드 공고 101개**에서 현업 AI 활용의 단서를 찾을 수 있습니다.

### 공고에서 드러나는 현업 AI 활용 패턴

**패턴 1: AI를 개발 도구로 쓰는 회사 (8개)**

> "Claude Code, Codex 등 LLM 도구를 개발 과정에서 적극적으로 활용" — 소프트스퀘어드
> "AI 도구 마스터리: Cursor/Claude/Gemini/MCP 활용" — 메라키플레이스
> "AI 기반의 코딩 도구를 적극 활용하여 UI 컴포넌트 개발속도를 높이고" — 도이치모터스

이 회사들은 **이미 팀 전체가 AI 도구를 쓰고 있고**, 새로 뽑는 사람도 당연히 써야 한다는 입장입니다.

**패턴 2: AI 도구를 문화로 만드는 회사**

> "전통적인 개발 방식에 머무르지 않고 Claude, Cursor 등 최신 AI 도구를 적극적으로 활용하여 빠르고 효율적으로 문제를 해결할 수 있는 분" — 코워크위더스

"전통적인 개발 방식에 머무르지 않고" — 이 표현이 핵심입니다. AI 도구를 쓰는 걸 **선택이 아니라 문화**로 보고 있어요.

**패턴 3: AI 거부감 없는 사람을 찾는 회사**

> "Claude, Cursor 등 AI 도구를 활용해 개발 생산성을 높여본 경험이 있거나, **이에 대한 거부감이 없으신 분**" — 코워크위더스
> "AI 발전 속도를 이해하고 자신의 역량으로 흡수할 수 있는 분" — 슈퍼센트

아직 AI를 본격적으로 안 쓰고 있지만, **"적어도 거부하지는 마라"**는 시그널입니다.

### 국내 기업 기술 블로그에서 본 AI 활용 사례

국내 테크 기업들도 기술 블로그에서 AI 활용 사례를 점점 더 공유하고 있습니다:

- **우아한형제들** (techblog.woowahan.com) — GenAI SDK 자체 개발, LLMOps 플랫폼 구축
- **토스** (toss.tech) — AI 기반 자동화, 코드 품질 관리에 AI 적용
- **카카오** (tech.kakao.com) — AI/ML 파이프라인, 추천 시스템
- **당근** (medium.com/daangn) — 70명+ FE 조직, 검색/추천에 AI 적용
- **네이버** (d2.naver.com) — HyperCLOVA, AI 검색, 코드 자동 생성

이 회사들의 기술 블로그를 정기적으로 읽는 것만으로도, **현업에서 AI가 어떤 맥락으로 쓰이는지** 감을 잡을 수 있어요. 각 기업의 구체적인 AI 활용 사례는 별도 글에서 더 자세히 다루겠습니다.

## 4. "쓰긴 쓰는데, 신뢰는 반반" — 현업의 솔직한 온도

숫자만 보면 "와, 다 쓰네!" 싶지만, 현실은 좀 더 복잡합니다.

### AI를 쓰면서도 불안한 것들

Stack Overflow 설문에서 개발자들이 꼽은 **AI의 문제점**:

| 문제 | 응답률 |
|------|--------|
| 잘못된 정보/허위 정보 생성 | **79%** (윤리적 우려 1위) |
| 코드 출처/저작권 불분명 | **65%** |
| 복잡한 작업 처리 능력 부족 | **45%** (프로 개발자 기준) |
| 코드베이스 이해 부족 | 주요 도전 과제 |

**프로 개발자의 45%가 "복잡한 작업에는 AI가 아직 못한다"**고 답했습니다. 간단한 CRUD나 보일러플레이트는 잘 만들지만, 복잡한 비즈니스 로직이나 아키텍처 설계에서는 여전히 사람이 판단해야 한다는 거죠.

### 현업에서의 암묵적 규칙들

공고나 설문에는 안 나오지만, 현업 개발자 커뮤니티에서 자주 보이는 패턴:

1. **"AI가 짠 코드도 네 코드다"** — AI가 만든 코드의 버그도 본인 책임. 무조건 리뷰 필수.
2. **"복붙 금지, 이해 필수"** — AI 출력을 그대로 쓰면 시니어에게 지적당함. 왜 이렇게 짰는지 설명할 수 있어야 함.
3. **"보안 민감 코드는 AI에 넣지 마라"** — 회사 내부 코드를 외부 AI에 올리는 건 보안 이슈. 사내 정책 확인 필수.
4. **"AI로 만든 테스트는 한번 더 돌려봐라"** — AI가 만든 테스트 케이스가 실제로 통과하는지 확인해야 함.

## 5. 글로벌 트렌드: AI 코딩의 미래 방향

### GitHub Octoverse 2023 — 오픈소스와 AI

GitHub의 연간 리포트에서 주목할 데이터:

- **92%**의 개발자가 AI 코딩 도구를 사용 중이거나 실험 중
- 생성형 AI 관련 오픈소스 프로젝트가 **GitHub Top 10 인기 프로젝트**에 진입
- AI 관련 GitHub 활동이 폭발적으로 증가

> 📎 출처: GitHub Octoverse 2023 — The state of open source and rise of AI
> https://github.blog/news-insights/research/the-state-of-open-source-and-ai/

### 앞으로 1년 내 예상 변화

개발자들이 "향후 1년 내 AI가 더 통합될 것"이라고 답한 분야:

- **코드 문서화**: 81%
- **테스트 코드**: 80%
- **코드 작성**: 76%

코드를 "짜는 것"뿐만 아니라, **문서화와 테스트**에서 AI 활용이 급격히 늘어날 거라는 전망입니다. 이건 프론트엔드 개발자에게도 직접적인 영향을 미칩니다 — Storybook 문서화, Jest/Vitest 테스트 작성에 AI를 쓰게 될 거라는 뜻이니까요.

## 6. 취준생이 여기서 가져가야 할 것

### 현업의 AI 온도 요약

| 상태 | 비율 | 의미 |
|------|------|------|
| 적극 사용 중 | ~62% | 이미 일상이 됨 |
| 관심은 있음 | ~14% | 곧 도입 예정 |
| 안 쓰거나 부정적 | ~24% | 여전히 소수 존재 |

**다수의 현업 개발자가 이미 AI를 쓰고 있지만, 맹신하지는 않습니다.** "도구로서 활용하되, 결과는 내가 검증한다" — 이게 현업의 태도예요.

### 취준생에게 필요한 3가지 자세

**1. "쓸 줄 알기" — 기본 도구 사용 경험**
- Cursor로 코드 작성, Claude로 코드 리뷰 — 이 정도는 해봐야 합니다
- 포트폴리오에 "AI 도구 활용 경험" 한 줄 넣기

**2. "의심할 줄 알기" — AI 출력 검증 능력**
- AI가 만든 코드를 그대로 쓰는 게 아니라, **읽고 이해하고 수정하는 능력**
- 면접에서 "AI가 만든 코드에 버그가 있으면 어떻게 하세요?" 같은 질문에 대비

**3. "설명할 줄 알기" — AI 활용 과정을 문서화**
- "AI를 어떤 상황에 썼고, 어떤 부분은 직접 수정했는지" 설명할 수 있어야 합니다
- 이게 "AI에 의존하는 개발자"와 "AI를 도구로 쓰는 개발자"의 차이

## 마무리

**"현업에서는 AI를 진짜 쓰나요?"**

네, 62%가 쓰고 있고, 92%가 관심을 갖고 있습니다. 한국도 이미 채용공고에 AI 도구 경험을 요구하는 곳이 15%입니다.

하지만 현업의 태도는 **"맹신"이 아니라 "활용"**입니다. AI가 짜준 코드를 그대로 커밋하는 게 아니라, 검증하고 이해하고 수정해서 쓰는 거예요.

취준생에게 가장 중요한 건 이겁니다:

> **"AI를 쓸 줄 아는데, AI 없이도 할 줄 아는 사람"**

이 균형을 보여줄 수 있으면, 어떤 회사에서든 환영받을 겁니다.
