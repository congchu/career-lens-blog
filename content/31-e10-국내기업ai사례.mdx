## 들어가며

"현업에서 AI 진짜 쓰나요?" — 취준 커뮤니티에서 가장 많이 보이는 질문 중 하나입니다.

글로벌 설문([Stack Overflow 2024](https://survey.stackoverflow.co/2024/ai))에 따르면 개발자 62%가 이미 AI를 쓰고 있지만, 솔직히 이런 생각 들지 않나요?

> "그건 미국 얘기잖아. **한국 회사는** 어때?"

그래서 이번에는 **국내 테크 기업들이 기술 블로그에 직접 공개한 AI 활용 사례**를 모아봤습니다. 추측이 아니라, 기업들이 "우리는 이렇게 AI를 쓰고 있다"고 공개한 내용입니다.

## 1. 우아한형제들 (배달의민족) — "AI 프로덕트 10개, 프론트엔드도 만든다"

배민의 AI 활용은 기술 블로그에 가장 상세하게 공개되어 있어서, 제일 먼저 다룹니다.

### 무슨 일이 일어났나

배민 AI플랫폼팀에 따르면, **2024~2025년 사이에 AI 프로덕트가 10개 이상** 만들어졌습니다. 그런데 더 주목할 포인트는 이겁니다:

> "AI 프로덕트 개발 주체가 데이터 사이언티스트·머신러닝 엔지니어에서 **PM, 프론트엔드, 백엔드 엔지니어까지 확장**되었습니다."
> — 우아한형제들 기술블로그, 「LLMOps로 확장하는 AI플랫폼 2.0」

이게 무슨 뜻인지 아시겠어요? **프론트엔드 개발자가 AI 프로덕트를 직접 만드는 시대**가 이미 배민에서는 시작됐다는 겁니다. AI 팀만의 일이 아니에요.

### 구체적으로 뭘 만들었나: GenAI SDK

배민은 GPT, Claude, Gemini 등 여러 LLM을 동시에 쓰다 보니, 이런 문제가 생겼습니다:

- API 호출 방식이 벤더마다 다 다름 (Azure OpenAI ≠ Google Gemini ≠ AWS Bedrock)
- API 키를 벤더별로 따로 관리해야 함
- 프롬프트가 코드, 구글 시트, 파일에 흩어져 버전 관리 불가능
- 외부 API 장애 시 서비스 다운

그래서 자체 개발한 게 **GenAI SDK**입니다. 핵심 구조:

| 컴포넌트 | 역할 | 기반 기술 |
|----------|------|-----------|
| **GenAI SDK** | 어떤 LLM이든 같은 코드로 호출 | LiteLLM (오픈소스) |
| **프롬프트 관리** | 버전 관리, 협업, A/B 테스트 | Langfuse (오픈소스) |
| **API Gateway** | 키 관리, 비용 추적, 장애 대응 | 자체 구축 |
| **GenAI Studio** | 프롬프트 실험 UI | 자체 구축 |

코드로 보면 이 차이가 확 느껴집니다:

**기존 방식 — 벤더마다 다른 코드:**
```
# Azure OpenAI
client = boto3.client("bedrock-runtime", ...)
response = client.invoke_model(modelId="bedrock/claude-3", ...)

# Google Gemini  
client = genai.Client(api_key="...")
response = client.models.generate_content(model="gemini-2.5-flash", ...)
```

**GenAI SDK 적용 후 — 하나의 인터페이스:**
```
# 어떤 모델이든 같은 코드
response = client.completion(
    model="gpt-4o-mini",  # 모델명만 바꾸면 됨
    messages=[{"role": "user", "content": "Hello!"}],
)
```

### 프론트엔드 개발자에게 시사점

1. **LLM API를 직접 호출하는 게 아니라 SDK를 통해 사용** — 프론트엔드에서 AI 기능을 넣을 때도 이런 추상화 계층을 이해해야 함
2. **프롬프트 관리 = 코드 관리** — 프롬프트도 버전 관리하고, A/B 테스트하고, 배포하는 시대
3. **"AI 프로덕트를 만드는 프론트엔드"가 이미 현실** — 배민에서 이미 일어나고 있는 일

## 2. 당근 — "프론트엔드 70명이 쓰는 개발 환경"

### 한국 최대급 프론트엔드 조직

당근은 **70명이 넘는 프론트엔드 엔지니어**가 일하는 조직입니다. "프론트엔드코어" 팀이 전체 FE 엔지니어의 개발 환경을 만드는 역할을 합니다.

기술 블로그에서 공유하는 내용을 보면:

- **웹뷰 기반 서비스** — iOS/Android 웹뷰에서의 입력 경험 최적화
- **이벤트 소싱 아키텍처** — Node.js에서 Redux 패턴을 서버에 적용
- **대규모 모듈 관리** — 200MB 모듈을 팀 단위로 분할하는 Dynamic Feature Module

### AI와의 접점

당근 기술 블로그에서 AI 코딩 도구에 대한 직접적인 글은 아직 많지 않습니다. 하지만 70명 규모의 FE 조직에서 AI 도구 도입이 어떤 영향을 미칠지 생각해보면:

- **컴포넌트 생성 자동화** — 70명이 만드는 수천 개의 컴포넌트, AI로 보일러플레이트 생성
- **코드 리뷰 보조** — 대규모 조직에서 PR 리뷰 부담 경감
- **웹뷰 크로스 플랫폼 이슈** — iOS/Android 웹뷰 버그를 AI로 디버깅

> 📎 당근 기술블로그: https://medium.com/daangn

## 3. 토스 — "금융 서비스에서의 AI"

### 금융 + AI의 만남

토스는 금융 서비스라는 특성 때문에 AI 활용에 더 신중한 접근을 합니다. 보안과 정확성이 생명인 도메인이니까요.

토스 기술 블로그(toss.tech)에서는:

- **제로트러스트 보안 체계** — AI를 쓰더라도 보안 기준은 절대 타협하지 않는 문화
- **프론트엔드 성능 최적화** — 대규모 트래픽을 처리하는 금융 앱의 기술 깊이

### 프론트엔드 개발자에게 시사점

금융 도메인에서 AI를 쓸 때는 일반 서비스와 다른 고려사항이 있습니다:

- **민감 데이터를 외부 AI에 보낼 수 없음** — 사내 AI 인프라 필요
- **AI 출력의 정확성 검증 필수** — 금액 계산이나 약관 안내에서 AI 오류는 법적 문제
- **규제 준수** — 금융 AI 활용에 대한 금융감독원 가이드라인

> 📎 토스 기술블로그: https://toss.tech

## 4. 네이버 — "HyperCLOVA에서 자체 AI 생태계까지"

### 한국의 AI 선두 주자

네이버는 국내에서 AI에 가장 적극적인 기업입니다. HyperCLOVA를 자체 개발하고, 이를 기반으로 다양한 서비스에 AI를 적용하고 있어요.

네이버 D2(d2.naver.com)와 DEVIEW 발표에서 공유한 내용:

- **CLOVA Studio** — 자체 LLM 기반 개발 플랫폼
- **AI 코드 어시스턴트** — 자체 코딩 보조 도구 개발
- **스마트 에디터** — AI 기반 글쓰기 보조
- **검색 AI** — 자연어 질의 기반 검색 결과 생성

### 프론트엔드 개발자에게 시사점

네이버급 회사에서는 **외부 AI(GPT, Claude)가 아닌 자체 AI**를 쓰는 경우가 많습니다. 프론트엔드 개발자가 자체 AI API를 연동하는 건 외부 API를 쓰는 것과 크게 다르지 않지만, **사내 문서와 가이드를 이해하는 능력**이 더 중요해요.

> 📎 네이버 D2: https://d2.naver.com

## 5. 카카오 — "추천 시스템과 카카오브레인"

카카오는 AI/ML을 주로 **추천 시스템**에 활용하고 있습니다:

- **카카오톡 선물하기** — AI 기반 상품 추천
- **멜론** — 음악 추천 알고리즘
- **카카오맵** — 경로 최적화

카카오브레인(현 카카오)에서는 이미지 생성 AI(칼로)를 개발한 경험도 있어요. 프론트엔드 관점에서는 **추천 API를 호출하고 결과를 UI로 보여주는 작업**이 실제 업무가 됩니다.

## 취준생이 이걸 어떻게 활용할까?

### 1. 지원 전 해당 기업 기술 블로그 읽기

면접에서 가장 좋은 인상을 주는 방법:

> 면접관: "우리 기술에 대해 아는 거 있나요?"
> 지원자: "기술 블로그에서 GenAI SDK 개발기를 봤는데, LiteLLM 기반으로 멀티 프로바이더를 추상화한 접근이 인상적이었습니다."

이렇게 답할 수 있으면 **바로 다릅니다.**

### 2. 기술 블로그 읽기 루틴 만들기

매일 읽을 필요 없습니다. **일주일에 1~2개면 충분해요.**

**추천 방법:**
- **GeekNews** (news.hada.io) — 한국판 Hacker News, 국내외 기술 글 큐레이션
- **RSS 리더** — Feedly 등에 관심 기업 블로그 구독
- **Twitter/X** — 기업 기술 계정 팔로우

기술 블로그를 읽고 **자신만의 언어로 정리**하면, 그 자체가 포트폴리오가 됩니다.

> "배민의 GenAI SDK 아키텍처를 분석하고, 이를 참고해 개인 프로젝트에서 LLM API 추상화 레이어를 구현했습니다."

이 한 줄이 이력서에 있으면, 면접관은 "이 사람은 현업 기술을 따라가고 있구나"라고 판단합니다.

## 마무리

국내 테크 기업들의 AI 활용은 이미 **"쓸까 말까" 단계를 넘어 "어떻게 잘 쓸까" 단계**에 와 있습니다.

특히 배민의 사례가 보여주듯, AI 프로덕트 개발이 더 이상 AI 팀만의 일이 아닙니다. **프론트엔드 개발자도 AI 기능을 제품에 넣는 시대**가 이미 와 있어요.

취준생으로서 지금 할 수 있는 가장 효율적인 준비는:

> **1. 기술 블로그를 읽어서 현업 감각 쌓기**
> **2. AI 도구(Cursor, Claude)를 직접 써서 경험 만들기**
> **3. 둘을 합쳐서 "현업 사례를 참고한 AI 활용 프로젝트" 만들기**

이 세 가지면 됩니다.
