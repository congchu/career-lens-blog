## 들어가며

채용공고를 보다 보면 이런 구분이 있습니다.

> **자격요건**: React, TypeScript 경험 3년 이상
> **우대사항**: Next.js, 테스트 코드 작성 경험

취준생들이 흔히 하는 해석:

- 자격요건 = **필수**. 없으면 지원 불가.
- 우대사항 = **있으면 좋고 없어도 OK**.

**이 해석, 절반만 맞습니다.**

공고 101개를 전부 까봤더니, "우대사항"의 진짜 의미는 생각보다 복잡했어요.

---

## 먼저 충격적인 사실부터

### 원티드 프론트엔드 공고 101개 중, "우대사항"을 따로 분리한 공고는 5개뿐입니다.

| 구분 | 공고 수 |
|------|---------|
| 자격요건만 있고 우대사항 구분 없음 | **96개 (95%)** |
| 자격요건 + 우대사항 명확히 분리 | **5개 (5%)** |

**95%의 공고가 자격요건에 모든 걸 몰아넣고 있습니다.**

이게 뭘 의미하냐면:

1. "우대사항"이라는 칸 자체가 없는 공고가 대부분
2. 자격요건에 "필수"와 "있으면 좋은 것"이 섞여 있음
3. **취준생이 "이건 우대니까 없어도 되지?"라고 판단할 근거 자체가 없음**

---

## 그러면 자격요건에 써있는 건 전부 필수인가요?

아닙니다. 자격요건에 있더라도 **진짜 필수 / 사실상 우대 / 희망사항**이 섞여 있어요. 구분하는 법을 알려드릴게요.

### 패턴 1: 진짜 필수 — "경험 N년 이상"

> "React 기반 프로젝트 개발 경험 (3년 이상)"
> "TypeScript 사용 경험"

이런 건 진짜 필수에 가깝습니다. 없으면 서류에서 걸러질 확률이 높아요.

**판별 키워드**: "경험 N년 이상", "숙련된", "능숙한", "필수"

### 패턴 2: 사실상 우대 — "~해본 경험이 있으신 분"

> "테스트 코드 작성 경험이 있으신 분"
> "CI/CD 파이프라인 구축 경험"
> "모노레포 환경 경험"

이건 **"있으면 좋겠지만 없다고 탈락시키지는 않겠다"**는 뜻인 경우가 많습니다.

**판별 키워드**: "경험이 있으신 분", "해보신 분", "관심이 있는 분"

### 패턴 3: 희망사항 — 문화 핏

> "새로운 기술과 도전에 열린 태도를 가진 분"
> "원활한 협업과 커뮤니케이션 역량을 갖춘 분"
> "주도적으로 문제를 해결하는 분"

이건 자격요건이 아니라 **"우리는 이런 사람을 좋아해요"**입니다. 이력서에 증명할 수도 없고, 면접에서 보는 부분이에요.

---

## 실제 공고로 보는 3단계 해석

하나의 공고를 예시로 3단계를 나눠볼게요. 씨엘모빌리티의 프론트엔드 공고입니다:

> **자격요건:**
> - 새로운 기술과 도전에 열린 태도를 가진 분 → **희망사항** (문화 핏)
> - React, Vue.js 등 최신 프론트엔드 프레임워크 경험 → **진짜 필수**
> - 원활한 협업과 커뮤니케이션 역량 → **희망사항** (문화 핏)

전부 "자격요건"에 들어있지만, **실제로는 1개만 기술적 필수**이고 나머지는 희망사항이에요.

---

## 우대사항을 따로 쓴 5개 공고는 뭐가 달랐나?

우대사항을 명확히 분리한 공고들을 살펴봤습니다.

### 도비스튜디오

> **자격요건**: React, TypeScript 경험
> **우대**: "Atomic Design 기반 프로젝트 구조를 설계·운영해본 경험", "복잡한 상태 관리(Server/Client State 분리) 경험", "React Query 등을 활용해 비동기 데이터를 효율적으로 처리할 수 있는 분"

### 볼트테크코리아

> **우대**: "한국어 및 영어 능통자 (추가 외국어 능력 보유 시 우대)", "Scrum 방법론을 사용하는 애자일 팀 환경 경험"

패턴이 보이시나요?

**우대사항에 들어가는 것들:**
- 특정 아키텍처 경험 (Atomic Design, 모노레포)
- 고급 상태관리 경험 (Server/Client State 분리)
- 언어 능력
- 방법론 경험 (Scrum, Agile)
- 특정 도구 심화 (React Query, TanStack)

**자격요건에 남는 것들:**
- React, TypeScript — 프레임워크/언어 기본기
- 협업 경험
- N년 이상 경력

**즉, 우대사항 = "이 팀에서 특별히 중요한 심화 역량"입니다.**

---

## 그래서 신입은 어떻게 판단해야 하나요?

### 1단계: 기술 키워드를 먼저 확인

공고에서 기술 키워드를 뽑아서, 내가 몇 개나 매칭되는지 세보세요.

원티드 프론트엔드 공고 기준 **가장 많이 등장하는 기술**:

| 키워드 | 자격요건 등장 수 |
|--------|----------------|
| React | 65개 (64%) |
| TypeScript | 40개 (40%) |
| Next.js | 28개 (28%) |
| CSS/스타일링 | 29개 (29%) |
| Git | 28개 (28%) |
| JavaScript | 24개 (24%) |

**React + TypeScript**가 매칭되면, 이미 자격요건의 핵심은 통과입니다.

### 2단계: "경험" 키워드의 강도 판단

| 표현 | 강도 | 없으면? |
|------|------|---------|
| "필수", "N년 이상" | 강 | 서류 탈락 가능 |
| "경험이 있으신 분" | 중 | 없어도 다른 걸로 커버 가능 |
| "관심이 있는 분" | 약 | 거의 영향 없음 |
| "열린 태도" 등 성향 | 없음 | 이력서로 증명 불가 |

### 3단계: 70% 규칙

자격요건의 기술 키워드 중 **70% 이상** 매칭되면 지원해도 됩니다.

100% 매칭될 필요 없어요. 왜냐하면:

1. **95%의 공고가 자격과 우대를 섞어 써놓았으니까** — 자격요건에 있어도 실제로는 "있으면 좋은 것"일 수 있음
2. **회사도 100% 매칭을 기대하지 않으니까** — 모든 조건을 만족하는 사람은 이미 시니어급
3. **신입에게 기대하는 건 "기본기 + 성장 가능성"이니까**

---

## 가장 많이 하는 실수

### 실수 1: "우대사항이니까 준비 안 해도 되지"

**틀렸습니다.** 우대사항은 "없어도 되는 것"이 아니라 **"있으면 확실히 유리한 것"**이에요.

같은 조건의 두 후보가 있을 때, 우대사항을 충족하는 사람이 뽑힙니다. 경쟁이 치열한 시장에서 우대사항은 사실상 **당락을 가르는 차별화 포인트**예요.

### 실수 2: "자격요건 하나라도 없으면 지원 안 해"

React, TypeScript, Next.js, GraphQL, AWS, Docker, CI/CD, 테스트 코드...

이걸 전부 갖춘 신입이 있을까요? **없습니다.** 회사도 알아요. 자격요건은 "이상적인 후보"를 그린 것이지, "최소 요건"이 아닙니다.

### 실수 3: "자격요건 = 면접 질문 범위"라고 착각

자격요건에 "Zustand 경험"이 있다고 면접에서 Zustand만 물어보진 않아요. 자격요건은 **업무에 쓰는 기술**이지, **면접 출제 범위**가 아닙니다. React 기초, JavaScript 동작 원리 등 기본기를 더 많이 물어봐요.

---

## 우대사항 대비 전략

### 지금 당장 할 수 있는 것

1. **테스트 코드 경험 만들기** — Jest/Vitest + Testing Library로 사이드 프로젝트에 테스트 추가. 공고에서 자주 보이는 우대 키워드
2. **CI/CD 경험** — GitHub Actions로 자동 배포 파이프라인 구축. 30분이면 됨
3. **AI 도구 활용** — 15%의 공고가 AI 도구 경험을 요구하기 시작. Cursor/Claude 써보기

### 이력서에 녹이는 법

우대사항 기술이 있다면 이렇게 표현하세요:

- ❌ "Jest 사용 가능"
- ✅ "Vitest + React Testing Library로 핵심 컴포넌트 24개 테스트 커버, 배포 전 자동 검증 파이프라인 구축"

구체적인 숫자와 맥락이 있으면, 우대사항이 **합격의 결정타**가 됩니다.

---

## 정리

| 오해 | 현실 |
|------|------|
| 자격요건 = 필수 | 95%의 공고가 필수+우대를 섞어 놓음 |
| 우대사항 = 없어도 OK | 경쟁에서 당락을 가르는 차별화 포인트 |
| 100% 매칭해야 지원 가능 | 70% 매칭이면 충분히 도전 가치 있음 |
| 자격요건 전부 면접에 나옴 | 기본기 질문이 더 많음 |

**채용공고의 "자격요건"과 "우대사항"은 칼로 자르듯 나뉘는 게 아닙니다.** 특히 원티드에서는 95%의 공고가 구분조차 하지 않아요.

중요한 건 이겁니다:
- **핵심 기술** (React, TypeScript)은 반드시 있어야 하고
- **나머지는 많을수록 유리**하되, 없다고 포기하지는 말 것
- **우대사항은 "보너스"가 아니라 "경쟁력"**이라고 생각할 것
