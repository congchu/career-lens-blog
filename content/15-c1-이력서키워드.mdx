---

## 이력서를 쓸 때 가장 막막한 순간

기술스택을 나열하는 칸 앞에서 멈춰본 적 있나요?

"React는 당연히 쓰고... TypeScript도 넣고... AWS는 좀 아는데 넣어도 되나? 테스트 코드는 찍먹 수준인데..."

감으로 채우는 이력서는 불안합니다. 그래서 **채용공고 101개의 자격요건과 우대사항을 전부 분석**해서, **실제로 회사들이 원하는 키워드**를 정리했어요.

---

## 📊 자격요건(필수)에 가장 많이 등장하는 기술

"이건 없으면 서류에서 걸릴 수 있다"는 키워드들입니다.

| 순위 | 키워드 | 자격요건 등장 | 비율 |
|------|--------|-------------|------|
| 1 | **React** | 60개 | 59% |
| 2 | **TypeScript** | 32개 | 31% |
| 3 | **Next.js** | 28개 | 27% |
| 4 | HTML/CSS | 22개 | 21% |
| 5 | JavaScript | 17개 | 16% |
| 6 | Git | 17개 | 16% |
| 7 | React Query | 12개 | 11% |
| 8 | Node.js | 10개 | 9% |
| 9 | Vue | 8개 | 7% |
| 10 | AWS | 8개 | 7% |

**React(59%)는 논쟁의 여지 없는 1위.** 프론트엔드 이력서에 React가 없으면 절반 이상의 공고에서 자격요건을 못 채우는 겁니다.

TypeScript(31%)와 Next.js(27%)가 그 뒤를 따라요. 이 세 가지는 **2026년 프론트엔드의 기본 3종 세트**라고 봐도 됩니다.

---

## 📊 우대사항에 가장 많이 등장하는 키워드

"없어도 지원은 되지만, 있으면 확실히 유리한" 키워드들입니다.

| 순위 | 키워드 | 우대사항 등장 | 비율 |
|------|--------|-------------|------|
| 1 | **배포/운영 경험** | 40개 | 39% |
| 2 | **AI/LLM 활용** | 27개 | 26% |
| 3 | **성능 최적화** | 25개 | 24% |
| 4 | AWS | 17개 | 16% |
| 5 | CI/CD | 15개 | 14% |
| 6 | **디자인 시스템** | 14개 | 13% |
| 7 | **테스트 코드** | 12개 | 11% |
| 8 | Next.js | 11개 | 10% |
| 9 | Docker | 7개 | 6% |
| 10 | Jest | 7개 | 6% |

여기서 주목할 건 **필수에는 별로 안 나오는데 우대에서 많이 나오는 키워드**예요.

---

## 🔥 핵심 발견: 필수 vs 우대 갭

| 키워드 | 필수 | 우대 | 해석 |
|--------|------|------|------|
| **성능 최적화** | 4개 | **25개** | 할 줄 알면 엄청난 차별화 |
| **AI/LLM** | 12개 | **27개** | 2026년 가장 핫한 우대사항 |
| **디자인 시스템** | 0개 | **14개** | 필수는 아닌데 있으면 사랑받음 |
| **테스트 코드** | 2개 | **12개** | 신입이 하면 "오?" 반응 |
| **CI/CD** | 6개 | **15개** | DevOps 감각 어필 가능 |

이게 무슨 의미냐면:

- **필수 기술** = 서류 통과의 최소 조건 (없으면 탈락)
- **우대 기술** = 같은 조건이면 이 사람을 뽑겠다는 기준

특히 **성능 최적화**는 필수에 4번밖에 안 나오지만 우대에 25번 나와요. 이력서에 "렌더링 최적화로 LCP를 2초에서 0.8초로 개선" 같은 한 줄이 있으면 면접관 눈이 갈 수밖에 없습니다.

## 신입 이력서 체크리스트

데이터를 기반으로, 신입 프론트엔드 이력서에 들어가야 할 것들을 정리했어요.

### 🟢 필수 (없으면 서류 탈락 위험)

- [ ] **React** — 59%가 필수로 요구. 컴포넌트 설계, 상태관리, 훅 활용
- [ ] **TypeScript** — 31%가 필수. 타입 정의, 제네릭 기본, any 남발 금지
- [ ] **Next.js** — 27%가 필수. SSR/SSG 차이, App Router 기본
- [ ] **HTML/CSS** — 21%가 필수. 시맨틱 마크업, Flexbox/Grid
- [ ] **Git** — 16%가 필수. 브랜치 전략, PR 경험

### 🟡 강력 추천 (있으면 서류 통과율 확 올라감)

- [ ] **React Query / 상태관리** — 서버 상태 vs 클라이언트 상태 구분
- [ ] **REST API 연동** — fetch/axios, 에러 처리, 로딩 상태
- [ ] **배포 경험** — Vercel, Netlify 등 실제 배포 + 도메인 연결
- [ ] **반응형** — 모바일 퍼스트, 미디어 쿼리

### 🔵 차별화 (우대사항에서 뽑은 킬러 키워드)

- [ ] **성능 최적화** — Lighthouse 점수 개선, 코드 스플리팅, 이미지 최적화
- [ ] **테스트 코드** — Jest + React Testing Library, 최소 1개 프로젝트
- [ ] **CI/CD** — GitHub Actions로 자동 배포 파이프라인
- [ ] **AI 도구 활용** — Copilot, Cursor 활용 경험 (26%가 우대!)
- [ ] **디자인 시스템** — Storybook으로 컴포넌트 문서화

## 이력서 작성 팁 3가지

### 1. 자격요건 키워드를 이력서에 그대로 넣어라

채용 담당자는 하루에 수십 개의 이력서를 봅니다. **자격요건에 적힌 키워드가 이력서에 보이면** 눈이 멈춰요.

공고에 "React, TypeScript 기반 서비스 개발 경험"이라고 써있으면, 이력서에도:

- ❌ "프론트엔드 개발을 했습니다"
- ✅ "**React + TypeScript** 기반 서비스를 개발하고 Vercel에 배포했습니다"

### 2. 우대사항 키워드는 "경험"으로 포장해라

우대사항에 "성능 최적화 경험"이라고 써있다면:

- ❌ 이력서에 "성능 최적화 가능"이라고 쓰기
- ✅ "Lighthouse 성능 점수를 **45 → 92로 개선**한 경험" 구체적 수치와 함께

### 3. 프로젝트 설명에 "왜"를 넣어라

- ❌ "Next.js로 블로그를 만들었습니다"
- ✅ "SEO와 초기 로딩 속도가 중요해서 **Next.js SSG를 선택**했고, 빌드 타임을 ISR로 최적화했습니다"

"왜 이 기술을 선택했는지"를 설명할 수 있는 사람은 신입이라도 다르게 보입니다.

---

## 2026년 주목할 키워드: AI/LLM

우대사항 2위가 **AI/LLM 활용(26%)**이에요. 이건 작년까지만 해도 없던 항목입니다.

실제로 공고에 이렇게 적혀있어요:

> "AI 도구 활용 생산성 개선 경험" — 윙잇
>
> "LLM을 활용한 애플리케이션 개발 또는 프롬프트 엔지니어링 경험" — 미소

프론트엔드 개발자에게 AI를 요구하는 시대가 왔습니다. Copilot이나 Cursor를 써본 경험, ChatGPT로 코드 리뷰를 한 경험 — 이런 것들을 이력서에 자연스럽게 녹여보세요.

## 정리: 이력서 우선순위 피라미드

```
         🔺 AI/LLM 활용
        🔺🔺 성능 최적화, 테스트, CI/CD
      🔺🔺🔺 배포 경험, 상태관리, API 연동
    🔺🔺🔺🔺 Next.js, HTML/CSS, Git
  🔺🔺🔺🔺🔺 React + TypeScript (기본 중의 기본)
```

아래층부터 채우세요. 기본이 없으면 위층이 의미가 없습니다.
기본이 있으면, 위층 하나만 더 있어도 확실히 달라집니다.
