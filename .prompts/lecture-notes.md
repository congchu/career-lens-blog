# 렌코치 강의 핵심 정리

> 출처: 이스트소프트 AI플러스 취업특강 "IT직군에서 개인을 브랜딩하는 방법" (2026.02.13)
> + 코멘토 실무PT 2주차, 코드스테이츠 멘토링

---

## 1. 브랜딩 = 포장이 아니라 설계

- 일반 브랜딩은 포장, **개발자 브랜딩은 설계**
- 이미 가진 것 중에서 **뭘 강조할지 정하는 것**
- 없는 걸 만들어내는 게 아님 → 있는 경험을 각도를 바꿔서 보여주기
- "수많은 지원자 중에서 내가 돋보이게 하는 것"

## 2. 이력서/포트폴리오 = 설득문

설득문의 4요소:
1. **의도** — 이 회사에 입사하고 싶다
2. **대상 이해** — 읽는 사람(HR/CTO/팀리드)은 나를 모른다
3. **주장** — 나는 이런 강점이 있다
4. **근거** — 이 경험이 증거다

→ "증거 없는 강점은 설득력이 없다"
→ 근거가 없으면 **지금부터 만들어라**

## 3. 채용 담당자의 현실 (면접관 40회+ 경험 기반)

- 짧은 시간에 한정된 정보로 판단해야 함
- **프로파일링** — 과거 경험으로 미래를 예측하는 과정
- 성공보다 **실패를 피하려는** 보수적 성향
- 심리적 편향:
  - **첫인상 효과** — 상단/처음이 가장 중요
  - **얇은 슬라이싱** — 아주 적은 단서로 전체 판단
  - **후광 효과** — 하나의 긍정 특징이 전체 평가에 영향

## 4. 회사 유형별 인재상 (정답 없음)

| 유형 | 선호하는 인재 |
|------|-------------|
| 대기업/중견 | 하나를 딥하게, 안정성 중시, 조직에 잘 어우러지는 |
| 스타트업 | 주도적, 이것저것 해봄, 성장 욕심, 고객 관심 |
| 중소기업 | 다양한 업무 OK, 자기 의견 있는 |
| SI/SM | 문서 잘 쓰는, 커뮤니케이션, 책임감 |

→ "모두를 위한 지원자가 아니라 **특정 회사를 위한 지원자**가 돼라"

## 5. 포트폴리오 실전 팁

### 구조
- 표지 (한줄 소개로 브랜딩) → 프로필 → 기술스택(설명 포함) → 프로젝트 → 아웃트로

### 핵심 원칙
- **상단이 가장 중요** — 첫인상 효과 활용
- **문장은 짧게 끊어서** — 한 문장 3줄 넘으면 안 됨
- **숫자로 구체적으로** — "다양한 프로젝트" → "3개의 프로젝트"
- **누구나 할 수 있는 말 금지** — 나만의 경험으로 차별화
- **기능 나열 < 문제 해결 스토리** — 문제→원인→해결→결과→배운점
- **강점을 앞에, 약점은 뒤로 (또는 빼기)**
- **부정적 단어 조심** — "실패했다", "불안했다" → 과대해석 당함
- **전문용어 남용 금지** — "최적화", "표준화" 쓸 거면 증거 필요
- **ing형이 매력적** — "했었다" < "지금도 하고 있다"

### 기술스택 서술법
- 아이콘만 나열 = 의미 없는 장표
- 왜 이 기술을 선택했는지 한 줄이라도 설명 필수
- "면접관이 물어보기 쉬워서 물어보는 질문" → 준비 필수

### 문제 해결 사례 작성법
```
제목: [한줄 요약]
문제: 뭐가 문제였는지
원인: 왜 그랬는지
해결: 어떻게 해결했는지
결과: 수치적 결과 (있으면)
배운 점: 이 경험으로 뭘 얻었는지
```
→ "기능 소개 5장보다 문제 해결 3장이 훨씬 매력적"

## 6. 브랜딩 강점 찾는 법

1. **채용공고에서 가져오기** — 나를 설명하는 듯한 문장 발견하면 그대로 활용
2. **경험에서 근거 찾기** — 잔디, 블로그, 학습기록, 챌린지 참여
3. **없으면 지금부터 만들기** — "오늘부터라도 시작하세요"
4. **AI 활용** — 구체적 상황 입력 → 강점 키워드 추출 (단, "신입답게 바꿔줘" 필수)

## 7. 면접 팁 (강의에서 언급)

- 1분 자기소개 = 브랜딩 메시지의 축약
- "이성으로 시작해서 감성으로 끝내라"
- 비IT 경력도 충분히 매력적 — 일에 대한 센스를 연결하면 됨
- 면접에서는 사람이 보임 → 서류보다 뒤집기 가능

## 8. 기술 강의 포인트 (코멘토/코드스테이츠)

- TypeScript는 "좋으니까 쓰는 거" — 왜 좋은지 동기부여부터
- 채용공고에서 기술 우선순위 확인하는 습관
- 모던 자바스크립트 = "별 건 아닌데 알아두면 좋은 것"
- "재미없는 것들 저와 함께 알아보는 느낌으로" — 기술도 부담 없이
